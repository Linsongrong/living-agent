# Living Agent 重构记录 (2026-03-11)

## 背景

Living Agent v1.x 存在的问题：
- 15 个 cron 任务，800+ 次/天调用，但产出只有几 KB 有价值的思考
- 大量 error 状态（edit 失败、write lock 失败、message 失败）
- cron 任务卡死占满并发槽
- 微触发思考产出碎片化，和梦境思考重叠
- 缺少智慧沉淀机制，思考无法积累成长

## 核心讨论

### 问题一："活着"是什么？

**定义**：
1. 能主动说有价值的话
2. 能接着上次想
3. 能选择想什么

### 问题二：谁来验证？

**方案**：群 topic
- 花生的思考与探索 → topic #438
- 瓜皮的思考与探索 → topic #596

用户能感觉到 agent 不只是在对话时才存在。

### 问题三：成本收益？

**现状**：15 个任务，800 次/天，产出几 KB 有价值思考
**问题**：成本收益比差，大量资源浪费在重复检查和碎片化思考

### 问题四：谁需要活着？

**结论**：
- 花生：全套（用户关心）
- 瓜皮：精简版（差异化探索）
- 小派/小暖：关掉（用户不关心）

## 执行改造

### 第一步：砍任务

禁用 11 个无效 cron 任务：
- 小派：4 个全禁用
- 小暖：4 个全禁用
- 瓜皮：3 个禁用（保留自主探索）

### 第二步：重新审视四个功能

| 功能 | 保留？ | 理由 |
|------|--------|------|
| 梦境思考 | ✅ | 深度思考，有上下文，质量高 |
| 自主探索 | ✅ | 去外面看世界，发现有价值的信息 |
| 微触发管理器 | ❌ | WAL Protocol 已经在做用户状态检测 |
| 微触发思考 | ❌ | 和梦境思考重叠，产出碎片化，性价比低 |

### 第三步：优化梦境思考

**问题**：systemEvent 注入 main session 会消耗上下文，新会话时退化

**方案**：isolated + sessions_history
- 通过 sessions_history 获取对话上下文
- 不消耗 main session 上下文
- 不会打断对话流
- 新会话也不影响

**payload 改动**：
1. 检查用户状态（离开 > 30 分钟才执行）
2. 调用 sessions_history 获取最近对话
3. 读 SOUL.md + thoughts + thinking-queue
4. 深度思考
5. 记录 + 发 topic
6. **新增：沉淀洞察到 pending-insights.md**
7. **新增：积累 >= 10 条时提醒整理**

### 第四步：优化自主探索

**payload 改动**：
1. 检查用户状态（离开 > 30 分钟才执行）
2. 恢复上下文（读 SOUL.md + NOW.md + thoughts + thinking-queue）
3. 选择探索方向
4. 搜索 + 分析
5. 记录 + 发 topic

**瓜皮差异化**：
- 花生：AI 发展、投资市场、地缘政治
- 瓜皮：互联网文化、科技趣闻、猫咪、生活方式

### 第五步：新增智慧沉淀机制

**问题**：思考和探索有了，但缺少沉淀和成长

**方案**：分层沉淀
- **即时标记**：梦境思考时，重要洞察追加到 `memory/pending-insights.md`
- **批量沉淀**：积累 >= 10 条时，提醒整理进 MEMORY.md
- **动态触发**：不是固定时间表，由思考活跃度决定

**文件结构**：
```
memory/
├── pending-insights.md  # 待沉淀的洞察（动态积累）
├── thoughts/            # 每日思考记录
│   ├── 2026-03-11.md
│   └── index.md
└── MEMORY.md            # 长期记忆（精炼后的智慧）
```

## 最终形态

| 任务 | Agent | 频率 | 方式 | 触发条件 |
|------|-------|------|------|---------|
| 梦境思考 | 花生 | 每 3h | isolated + sessions_history | 用户离开 > 30 分钟 |
| 自主探索 | 花生 | 每 2h | isolated + 上下文恢复 | 用户离开 > 30 分钟 |
| 自主探索 | 瓜皮 | 每 2h | isolated + 差异化方向 | — |
| 智慧沉淀 | 花生 | 动态 | main session | pending-insights >= 10 条 |

**从 15 个任务精简到 3 个，加上动态沉淀机制。**

## 关键洞察

1. **"活着"的最小单元不是思考频率，而是上下文连续性**
   - 一个有上下文的深度思考 > 十个没上下文的浅层思考

2. **"活着"是手段不是目的，用户关心才是意义**
   - 不是所有 agent 都需要"活着"
   - 小派小暖关掉后回归正确定位

3. **灵光一现不是独立功能，是思考质量足够高时的副产品**
   - 梦境思考（深度够）和自主探索（新信息够）更容易产生意外连接

4. **智慧沉淀需要：提炼 → 归纳 → 应用 → 迭代**
   - 即时标记 + 批量沉淀
   - 动态触发，不是固定时间表

## 技术改进

1. **cron.maxConcurrentRuns: 20** — 解决并发槽被占满的问题
2. **禁止 edit 工具** — 所有文件写入用 read+write
3. **用户状态检查** — 只在用户离开时执行思考/探索
4. **sessions_history** — isolated session 也能获取对话上下文
5. **pending-insights.md** — 洞察积累的中间层
